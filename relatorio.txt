1. RESUMO DA IMPLEMENTACAO
================================================================================

Foi realizada a integracao completa com a API do SmartPOS para sincronizacao de
produtos e baixa automatica de estoque apos vendas no site.

Principais funcionalidades implementadas:
- Servico de comunicacao com SmartPOS API (listagem e atualizacao de estoque)
- Sistema de mapeamento entre Produtos do Site <-> Produtos SmartPOS
- Baixa automatica de estoque ao finalizar pedido no Checkout
- Painel Administrativo para gerenciar mapeamentos

================================================================================
2. ARQUIVOS CRIADOS
================================================================================

Abaixo a lista de arquivos criados e suas responsabilidades:

A. CORE / SERVICOS
------------------
1. src/lib/smartpos.ts
   - Contem a logica de comunicacao com a API externa (fetch)
   - Funcoes: getSmartPOSProducts, updateSmartPOSStock, saveProductMapping, getAllMappings
   - Gerencia a autenticacao via headers (X-Api-Key-Secret)

B. API ROUTES (BACKEND)
-----------------------
2. src/app/api/smartpos/sync/route.ts
   - GET: Busca produtos do SmartPOS para exibicao no admin
   - POST: Salva o mapeamento manual feito pelo admin

3. src/app/api/smartpos/stock/route.ts
   - POST: Atualiza estoque de um item especifico (uso geral)

4. src/app/api/smartpos/process-order/route.ts
   - POST: Recebe itens do carrinho, busca mapeamentos e baixa estoque
   - Endpoint seguro chamado pelo checkout (server-side logic)

5. src/app/api/smartpos/mappings/route.ts
   - GET: Retorna todos os mapeamentos salvos no Firestore

C. FRONTEND / ADMIN
-------------------
6. src/app/admin/smartpos/page.tsx
   - Nova pagina no painel administrativo
   - Permite listar produtos do SmartPOS
   - Interface visual para ligar (mapear) produtos SmartPOS com produtos do Site
   - Indica visualmente quais produtos ja estao vinculados

================================================================================
4. FLUXO DE FUNCIONAMENTO
================================================================================

1. Configuracao (Admin):
   - O administrador acessa /admin/smartpos
   - Clica em "Buscar Produtos SmartPOS"
   - Seleciona para cada item do SmartPOS qual o produto correspondente no site
   - O sistema salva esse vinculo no Firestore (collection 'product_mappings')

2. Venda (Cliente):
   - Cliente finaliza compra no Checkout
   - Pedido e salvo no Firestore
   - Sistema chama API /process-order
   - Para cada item, sistema verifica se existe mapeamento
   - Se existir, chama API SmartPOS (PUT /products/stock/{id}) removendo a quantidade
   - Estoque do SmartPOS e atualizado em tempo real

================================================================================
5. OBSERVACOES DE SEGURANCA
================================================================================

- As chaves de API (Secret) nao sao expostas no cliente (browser)
- Toda comunicacao com SmartPOS e feita via API Routes (Node.js environment)
- O checkout apenas envia os itens para a API interna, que trata a logica segura
