1. RESUMO DA IMPLEMENTACAO
================================================================================

A integracao SmartPOS foi expandida para incluir tanto o CONTROLE DE ESTOQUE
quanto o FLUXO DE PAGAMENTO no checkout.

Funcionalidades:
1. Sincronizacao de Estoque (Real): Baixa estoque na API SmartPOS ao vender.
2. Mapeamento de Produtos (Real): Painel admin para vincular produtos.
3. Pagamento (Simulado/UX): Modal de pagamento com QR Code Pix e instrucoes
   para cartao, integrando visualmente com o fluxo do usuario.

================================================================================
5. OBSERVACOES IMPORTANTES
================================================================================

A integracao de PAGAMENTO via API (comunicar com a maquininha fisica para
acender o visor ou gerar Pix dinamico no banco) requer documentacao especifica
dos endpoints de Transacao (ex: POST /transactions).

Como essa documentacao nao estava disponivel, foi implementada uma SIMULACAO
DE ALTA FIDELIDADE que resolve a experiencia do usuario (UX). Para tornar
o pagamento real (backend), basta substituir as chamadas simuladas no
PaymentModal.tsx pelas chamadas reais da API quando disponiveis.

O controle de ESTOQUE esta funcionando com a API real configurada.
