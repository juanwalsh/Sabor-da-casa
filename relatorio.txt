1. INTEGRAÇÃO SMARTPOS - CONTROLE DE ESTOQUE (FUNCIONANDO)
================================================================================

O sistema de estoque está 100% integrado com a API SmartPOS.

- **Fluxo**:
  1. O sistema busca produtos do SmartPOS (`GET /products`).
  2. No painel Admin (`/admin/smartpos`), você vincula os produtos do site.
  3. Agora, a baixa de estoque ocorre **automaticamente** assim que o pagamento é confirmado pelo Gateway (Mercado Pago).

================================================================================
2. SISTEMA DE PAGAMENTO REAL E AUTOMATIZADO (MERCADO PAGO)
================================================================================

Para resolver a necessidade de "Pagamento Real" e "Validação Automática", integramos o **Mercado Pago**.

- **Pix Dinâmico**: O sistema gera um QR Code único para cada pedido via API oficial do Mercado Pago.
- **Validação Automática (Webhook)**: 
  - O Mercado Pago detecta o pagamento instantaneamente.
  - Ele envia um "aviso" (Webhook) para o nosso servidor.
  - O sistema valida o pagamento, altera o status do pedido para "Confirmado" e dispara a baixa de estoque no SmartPOS.

- **Vantagem**: Você não precisa conferir o extrato do banco manualmente; o site libera o pedido sozinho.

================================================================================
3. FLUXO DE AUTOMAÇÃO (COMO TUDO FUNCIONA JUNTO)
================================================================================

1. **Cliente**: Finaliza o pedido e vê o QR Code real do Mercado Pago.
2. **Mercado Pago**: Processa o Pix e confirma o recebimento.
3. **Servidor (Webhook)**: Recebe a confirmação e faz duas coisas:
   - Marca o pedido como PAGO no banco de dados (Firebase).
   - Envia o comando para o **SmartPOS** baixar o estoque dos produtos vendidos.
4. **Cozinha/Logística**: Recebe o pedido já confirmado e com estoque garantido.

5. CONFIGURAÇÕES NECESSÁRIAS NO .ENV.LOCAL
================================================================================

Para o funcionamento total, as seguintes chaves são obrigatórias:
- `SMARTPOS_API_KEY_SECRET`: Para o estoque.
- `MERCADO_PAGO_ACCESS_TOKEN`: Para os pagamentos reais.
