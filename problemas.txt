Checklist Completo - Nível Sênior

  1. TESTES (Prioridade Máxima)

  1.1 Testes Unitários (Vitest)

  □ Configurar Vitest
  □ Testar funções utilitárias (formatPrice, cn, etc)
  □ Testar Zod schemas
  □ Testar stores Zustand
  □ Testar hooks customizados
  □ Coverage mínimo: 70%

  1.2 Testes de Componentes (Testing Library)

  □ Configurar React Testing Library
  □ Testar componentes de UI isolados
  □ Testar formulários (checkout, login, produto)
  □ Testar interações (adicionar ao carrinho, etc)

  1.3 Testes E2E (Playwright)

  □ Configurar Playwright (já tem playwright.config.ts)
  □ Fluxo: Navegar cardápio → Adicionar ao carrinho → Checkout
  □ Fluxo: Login admin → Criar produto → Ver no cardápio
  □ Fluxo: Login admin → Ver pedidos → Atualizar status
  □ Testar responsividade (mobile/desktop)
  □ Testar dark mode

  1.4 Testes de API (Supertest ou Playwright)

  □ Testar POST /api/auth (login válido/inválido)
  □ Testar GET /api/products (lista produtos)
  □ Testar POST /api/orders (criar pedido)
  □ Testar middleware de auth (401 sem token)
  □ Testar rate limiting (429 após limite)

  ---
  2. CI/CD (Automação)

  2.1 GitHub Actions

  □ Workflow: Rodar testes em cada PR
  □ Workflow: Lint + Type check
  □ Workflow: Build de verificação
  □ Workflow: Deploy automático para Vercel (staging)
  □ Workflow: Deploy para produção (manual approval)

  2.2 Quality Gates

  □ PR só pode ser mergeado se testes passarem
  □ PR só pode ser mergeado se lint passar
  □ PR só pode ser mergeado se build passar
  □ Configurar branch protection rules

  ---
  3. OBSERVABILIDADE (Monitoramento)

  3.1 Error Tracking

  □ Configurar Sentry
  □ Capturar erros de frontend
  □ Capturar erros de API
  □ Alertas por email/Slack quando erro crítico
  □ Source maps para debug em produção

  3.2 Logging Estruturado

  □ Implementar logger (pino ou winston)
  □ Logs com correlation ID por request
  □ Logs de ações importantes (pedido criado, login, etc)
  □ Não logar dados sensíveis (senhas, tokens)

  3.3 Analytics/Métricas

  □ Tempo de resposta das APIs
  □ Taxa de erro por endpoint
  □ Produtos mais vendidos
  □ Horários de pico
  □ Funil de conversão (cardápio → carrinho → checkout → pedido)

  ---
  4. PERFORMANCE

  4.1 Frontend

  □ Implementar Suspense boundaries
  □ Lazy loading de componentes pesados
  □ Otimizar imagens (já usa next/image, mas verificar)
  □ Implementar skeleton loading em todas as listas
  □ Cache de dados com SWR (já usa, verificar configuração)
  □ Lighthouse score > 90 em todas as métricas

  4.2 Backend

  □ Cache de produtos (Redis/Upstash)
  □ Índices no Firestore para queries frequentes
  □ Paginação na listagem de pedidos
  □ Compressão de respostas (gzip)

  ---
  5. SEGURANÇA AVANÇADA

  □ Headers de segurança (CSP, HSTS, etc) - middleware
  □ Sanitização de inputs (XSS prevention)
  □ Validação de origem de requests (CORS)
  □ Audit log de ações administrativas
  □ Rotação de JWT_SECRET (documentar processo)
  □ Backup automático do Firestore
  □ Política de senha forte para admin

  6. DOCUMENTAÇÃO

  □ README completo com setup local
  □ Documentação de API (Swagger/OpenAPI)
  □ Documentação de arquitetura (diagrama)
  □ Guia de contribuição (CONTRIBUTING.md)
  □ Changelog (CHANGELOG.md)
  □ Documentação de variáveis de ambiente
  □ Runbook para incidentes

  7. CÓDIGO LIMPO

  8. FEATURES AVANÇADAS (Diferencial)

  □ Webhooks para notificar sistemas externos
  □ Exportar relatórios (PDF/Excel)
