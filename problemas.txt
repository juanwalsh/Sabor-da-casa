Melhorias Recomendadas - UX e Código

  Prioridade Alta (Impacto Direto no Usuário)

  1. Logout não limpa cookie do servidor

  Arquivo: src/store/authStore.ts:48-50

  Problema: O logout só limpa o estado local. O cookie admin_token permanece válido.

  Impacto: Usuário faz logout, fecha aba, reabre → ainda está logado.

  Correção:
  logout: async () => {
    await fetch('/api/auth', { method: 'DELETE' });
    set({ user: null, isAuthenticated: false });
  },

  ---
  2. Erros de validação não mostrados no checkout

  Arquivo: src/app/checkout/page.tsx

  Problema: Campos obrigatórios não mostram erro visual quando vazios. Só nome, telefone e email têm {errors.X}.

  Impacto: Usuário clica "Confirmar" e nada acontece - sem feedback.

  Correção: Adicionar mensagens de erro para todos os campos:
  <Input {...register('street')} />
  {errors.street && (
    <p className="text-sm text-destructive mt-1">{errors.street.message}</p>
  )}

  ---
  3. Cardápio usa dados locais, não Firebase

  Arquivo: src/app/cardapio/page.tsx:17

  import { products, categories, formatPrice } from '@/data/mockData';

  Problema: Cardápio mostra dados estáticos do mockData, não os produtos reais do Firebase.

  Impacto: Admin edita produto → cliente não vê alteração.

  Correção: Usar hook para buscar do Firebase:
  const { products, isLoading } = useProducts(); // fetch de /api/products

  ---
  4. Pedidos admin usa mock para nome de produto

  Arquivo: src/app/admin/pedidos/page.tsx:89-91

  const getProductName = (productId: string) => {
    return products.find((p) => p.id === productId)?.name || 'Produto não encontrado';
  };

  Problema: Busca nome do produto no mockData local, não no pedido salvo.

  Impacto: Se produto foi deletado, mostra "Produto não encontrado".

  Correção: O nome do produto já vem no pedido (order.items[].product.name), usar esse.

  ---
  5. Máscara de telefone inexistente

  Arquivo: src/app/checkout/page.tsx

  Problema: Campo de telefone aceita qualquer formato. Usuário digita 11999999999 sem formatação.

  Impacto: UX ruim + dados inconsistentes no banco.

  Correção: Usar react-input-mask ou custom hook:
  <Input
    {...register('phone')}
    placeholder="(11) 99999-9999"
    onChange={(e) => {
      const masked = maskPhone(e.target.value);
      setValue('phone', masked);
    }}
  />

  ---
  6. CEP não preenche endereço automaticamente

  Arquivo: src/components/ui/cep-validator.tsx

  Problema: Valida CEP mas não preenche rua/bairro/cidade automaticamente.

  Impacto: Usuário precisa digitar tudo manualmente.

  Correção: Usar ViaCEP API para auto-complete:
  const response = await fetch(`https://viacep.com.br/ws/${cep}/json/`);
  const data = await response.json();
  setValue('street', data.logradouro);
  setValue('neighborhood', data.bairro);
  setValue('city', data.localidade);

  Prioridade Média (Melhoria de Qualidade)

  7. Import não usado no upload

  Arquivo: src/app/api/upload/route.ts:4

  import mime from 'mime-types'; // Note: might not be installed...

  Problema: Import não é usado (mimeMap local resolve).

  Correção: Remover linha.

  ---
  8. Rating hardcoded nos cards de produto

  Arquivo: src/app/cardapio/page.tsx:400

  <span>4.9</span>

  Problema: Todos os produtos mostram 4.9 de avaliação.

  Correção: Ou remover o rating, ou implementar sistema de avaliações real.

  ---
  9. Mensagem de estoque insuficiente genérica

  Arquivo: src/app/api/orders/route.ts:125

  throw new Error(`Estoque insuficiente para ${item.product.name}`);

  Problema: Não diz quanto tem disponível.

  Correção:
  throw new Error(`Estoque insuficiente para ${item.product.name}. Disponível: ${currentStock}, Solicitado: ${item.quantity}`);

  ---
  10. Sem loading states consistentes

  Problema: Algumas páginas têm loading spinner, outras não.

  Correção: Criar componente <LoadingSpinner /> reutilizável e usar em todas as páginas que fazem fetch.

  ---
  11. Duplicação de stores (orderStore vs Firebase)

  Arquivos: src/store/orderStore.ts + src/store/orderHistoryStore.ts

  Problema: Dois stores diferentes para pedidos, podem ficar dessincronizados.

  Correção: Unificar em um único store que sincroniza com Firebase ou remover o store local.

  ---
  Prioridade Baixa (Nice to Have)

  12. Console.log em produção

  Arquivos: Vários

  console.log('Token de notificacao:', token);

  Correção: Remover ou usar logger condicional.

  ---
  13. Skeleton loading para imagens

  Problema: Imagens de produtos não têm placeholder enquanto carregam.

  Correção: Usar placeholder="blur" no next/image ou skeleton component.

  ---
  14. Sem confirmação ao cancelar pedido

  Problema: Admin pode cancelar pedido sem confirmação.

  Correção: Adicionar dialog de confirmação antes de cancelar.

  ---
  15. Falta acessibilidade em botões de quantidade

  Arquivo: src/components/menu/CartSidebar.tsx:127-146

  Problema: Botões +/- não têm aria-label.

  Correção:
  <button aria-label="Diminuir quantidade">
    <Minus />
  </button>

  Top 5 - Recomendação de Implementação

  1. Cardápio buscar do Firebase - Sem isso, o admin não serve para nada
  2. Erros de validação visíveis no checkout - Usuário não sabe o que está errado
  3. CEP com auto-complete - Reduz fricção significativamente
  4. Logout limpar cookie - Problema de segurança/UX
  5. Máscara de telefone - Dados consistentes
